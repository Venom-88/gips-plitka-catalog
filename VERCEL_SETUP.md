# Деплой «Гипс Стиль 31» на Vercel — пошагово (через панель)

Стек: Next.js 15 + Prisma + PostgreSQL. Сборка уже устойчива к отсутствию БД:
первый деплой откроется даже без Postgres (каталог пустой), после шага 4 всё заработает.

---

## Шаг 0. Что понадобится
- Аккаунт на **vercel.com** (у вас есть — команда «Stratton Oakmand»).
- Аккаунт **GitHub** (Vercel деплоит из репозитория).
- На компьютере — установленный **Node.js** (для одноразового наполнения БД).

---

## Шаг 1. Загрузить код на GitHub

Проект пока не в git. В папке `D:\GIPS-style` выполните в терминале:

```bash
git init
git add .
git commit -m "Гипс Стиль 31"
```

Создайте пустой репозиторий на github.com (кнопка **New**, без README), затем:

```bash
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/gips-style-31.git
git push -u origin main
```

> Файлы `.env`, `donor/`, `node_modules` не загрузятся — они в `.gitignore`/`.vercelignore`.

**Альтернатива без GitHub:** установить `npm i -g vercel`, в папке проекта выполнить `vercel` и следовать подсказкам. Тогда шаг 2 не нужен.

---

## Шаг 2. Импортировать проект в Vercel

1. Vercel → **Add New… → Project**.
2. **Import** ваш GitHub-репозиторий.
3. Framework Preset определится сам как **Next.js**. Ничего не меняйте.
4. Нажмите **Deploy**. Первый деплой пройдёт (сайт откроется, каталог пустой).

---

## Шаг 3. Создать базу данных (Postgres)

1. В проекте → вкладка **Storage** → **Create Database** → **Postgres** → **Continue**.
2. Имя любое (напр. `gips-db`), регион — ближайший (напр. Frankfurt). **Create**.
3. Vercel сам подключит базу к проекту и добавит переменные `POSTGRES_*`.
4. Откройте базу → вкладка **`.env.local`** (или **Quickstart**) и найдите строку
   **`POSTGRES_URL_NON_POOLING`** — скопируйте её значение (понадобится в шагах 4 и 5).

---

## Шаг 4. Переменные окружения

Проект → **Settings → Environment Variables**. Добавьте (Environment: **Production, Preview, Development** — можно все три):

| Name | Value |
|---|---|
| `DATABASE_URL` | вставьте значение `POSTGRES_URL_NON_POOLING` из шага 3 |
| `ENCRYPTION_KEY` | свой ключ (см. ниже) — НЕ из репозитория |
| `JWT_SECRET` | свой ключ (см. ниже) — НЕ из репозитория |
| `NEXT_PUBLIC_SITE_URL` | адрес сайта, напр. `https://gips-style-31.vercel.app` |

> ⚠️ `ENCRYPTION_KEY` и `JWT_SECRET` задавайте **свежими** значениями (в истории git лежат старые
> примеры — их использовать нельзя). Сгенерировать: `openssl rand -base64 48` (по одному на каждый).

Нажмите **Save**.

---

## Шаг 5. Создать таблицы и наполнить БД (один раз, с компьютера)

Prod-база пустая. Разверните схему и демо-данные, указав строку подключения из шага 3.

**PowerShell (Windows):**
```powershell
cd D:\GIPS-style
$env:DATABASE_URL="ВСТАВЬТЕ_POSTGRES_URL_NON_POOLING"
npx prisma migrate deploy
npm run db:seed
```

Создастся:
- 5 категорий, 15 товаров, работы, отзывы, контакты;
- админ: **admin@gips31.ru / admin12345** (смените пароль после входа).

---

## Шаг 6. Пересобрать деплой

Vercel → вкладка **Deployments** → у последнего деплоя меню **⋯ → Redeploy**
(чтобы применились переменные и появились данные). После сборки откройте адрес проекта —
каталог, товары и главная уже с контентом.

---

## Шаг 7. Загрузка фото (Vercel Blob) — опционально

Чтобы в админке работала кнопка **«Файл»** (прямая загрузка картинок):
1. **Storage → Create → Blob** → Create. Переменная `BLOB_READ_WRITE_TOKEN` добавится сама.
2. **Redeploy**.

До этого фото можно добавлять, вставляя **URL картинки** в поле изображения — это работает сразу.

---

## Шаг 8. Проверка
- `/` — главная, `/catalog` — каталог с фильтром, `/product/…` — товар.
- `/admin` — вход `admin@gips31.ru / admin12345` → **смените пароль** (раздел «Пользователи»).
- В админке → **Изображения** загрузите hero-фон и фото «О студии», выберите область кадра.
- Отправьте тестовую заявку → появится в админке (**Заявки**).
- `/sitemap.xml`, `/robots.txt` — отдаются.

---

## Шаг 9. Свой домен — опционально
Проект → **Settings → Domains** → добавьте домен, пропишите DNS по подсказке Vercel.
После этого обновите `NEXT_PUBLIC_SITE_URL` на новый адрес и **Redeploy**.

---

## ⚠️ 152-ФЗ (важно знать)
Vercel и его Postgres расположены **вне РФ**. По закону персональные данные граждан РФ должны
храниться на серверах в РФ. Для строгого соответствия — перед публичным запуском перенесите
хранение заявок (ПД) в РФ-хостинг, либо используйте это как демо/MVP. Заявки у нас шифруются
(AES-256), но требование о локализации это не заменяет.
