# Handoff: Главная страница сайта «Гипс Стиль 31» (мобильная версия)

## Overview
Это передача дизайна **мобильной главной страницы** сайта-каталога домашней студии гипсовой плитки ручной работы **«Гипс Стиль 31»** (г. Губкин). Главная решает задачи ТЗ: получение заявок/звонков, формирование доверия к небольшой студии, переход в каталог и захват контактов дизайнеров (B2B). Дизайн mobile-first, тёплый «ремесленный» premium-минимализм: графит + янтарь, фирменный мотив сот-гексагонов.

## About the Design Files
Файлы в этом пакете — это **дизайн-референсы, созданные в HTML** (прототип внешнего вида и поведения), **а не продакшн-код для прямого копирования**. Задача — **воссоздать эти макеты в реальной кодовой базе/CMS**, используя её принятые паттерны, библиотеки и компоненты. Если кодовой базы ещё нет, выбрать подходящий стек под цели проекта.

Прототип сделан как один HTML-файл (`Гипс Стиль 31 — Главная.dc.html`). Технически он использует внутренний рантайм песочницы (`support.js`, теги `<x-dc>`, `<helmet>`, веб-компонент `<image-slot>`) — **это служебные обёртки прототипа, их переносить не нужно**. Разработчику важна разметка секций, инлайновые стили, размеры, цвета и копирайт внутри.

Рекомендация по реализации (из ТЗ, п.11): готовая российская e-commerce CMS (1С-Битрикс / аналог) либо Tilda — в зависимости от бюджета; хостинг в РФ (152-ФЗ), HTTPS. Если делается на фронтенд-фреймворке — React/Next.js с server-side рендерингом для SEO.

## Fidelity
**High-fidelity (hifi).** Финальные цвета, типографика, отступы, состояния и копирайт. UI следует воссоздать пиксель-в-пиксель средствами целевого стека. Все `<image-slot>` — это **плейсхолдеры под реальные фото** (фактуры и интерьеры), которые заказчик предоставит; в проде это обычные `<img>`/фоновые изображения, заполняемые из админки.

## Viewport
- Дизайн-ширина контента: **390px** (iPhone-класс). Это базовый mobile-first брейкпоинт.
- Прототип показан в «телефоне»: внешняя серая подложка (`#DAD2C3`), колонка контента со скруглением 30px и фейковый статус-бар (`9:41` + иконки) — **это только презентационная рамка, в проде её нет**. Реализуется обычная адаптивная страница: контент-колонка тянется на 100% ширины вьюпорта на телефоне.
- Дальнейшие брейкпоинты (планшет/десктоп) в этом макете не заданы — нужно спроектировать отдельно (см. «Responsive»).

- **Якорные ссылки:** hero CTA → `#cat` (каталог) и `#form` (форма); B2B → `#form`. Плавный скролл.
- **Состояния кнопок:** `:active { transform: scale(.92–.98) }`; добавить `:hover` для десктопа (подъём/затемнение).
- **Инпуты:** `:focus` — рамка янтарная `#D99A2B`.
- **Форма:** валидация имени и телефона (маска `+7 (___) ___-__-__`), обязательный чекбокс согласия ПД; защита от спама (капча/honeypot — ТЗ п.11); отправка → e-mail + Telegram-уведомление владельцу (ТЗ п.12); экран «спасибо».
- **Калькулятор:** живой пересчёт при вводе площади.
- **image-slot → реальные фото:** в проде это изображения из админки/CMS; на работах накладывается водяной знак.

## State Management
Главная преимущественно статична. Динамика:
- Состояние формы заявки (поля, валидность, статус отправки, ответ сервера).
- Калькулятор: `площадь` → производные `упаковки`, `стоимость`.
- Корзина (глобальный стор): добавление по кнопке «+» на карточке товара, счётчик в шапке (на этом экране не показан, но предусмотреть).
- Данные товаров/категорий/отзывов — из CMS/API (не хардкод).

## Responsive
Текущий макет — только 390px (mobile-first). Спроектировать вверх:
- **Планшет (≥768):** категории и преимущества в 2–3 колонки; хиты — сетка вместо скролла; bottom bar можно скрыть, вернув звонок/мессенджеры в шапку.
- **Десктоп (≥1024):** контейнер ~1200px, hero на пол-экрана с текстом сбоку, многоколоночные сетки, мега-меню каталога, sticky-шапка с телефоном и мессенджерами. Bottom bar убрать; плавающие кнопки мессенджеров справа.
- Минимальный размер шрифта в интерфейсе — 12px; хит-таргеты ≥44px.

## SEO (из ТЗ, критично)
- Семантическая верстка, один H1 на страницу, иерархия H2/H3.
- Настраиваемые title/description/H1, ЧПУ, хлебные крошки, alt у всех изображений.
- Микроразметка Schema.org: Organization/LocalBusiness, Product (цена+наличие), BreadcrumbList, AggregateRating (отзывы).
- Перелинковка категорий в футере и блоках «похожие».
- sitemap.xml, robots.txt, изображения в WebP, lazy-loading, оптимизация скорости.
- Региональная привязка (Яндекс.Вебмастер) под Губкин и целевые города.
